# NeuroTest

Бекенд для сервиса по прохождению тестов с использованием LLM.

## Описание

NeuroTest — это API-сервис, который позволяет:
- Загружать файлы с тестами (в текущей версии поддерживается `.docx`, в планах — `.odt`, `.doc`, а также HTML и изображения);
- Автоматически парсить тесты и выделять вопросы с вариантами ответов (без правильных ответов);
- Генерировать правильные ответы на вопросы с помощью языковых моделей (Cerebras, OpenRouter);
- Получать список загруженных файлов по типам (исходные `.docx`, JSON с вопросами, JSON с ответами).

Проект построен на **FastAPI** с использованием **LangChain** для работы с LLM и **Pydantic** для валидации данных.  
В будущем планируется переход на **LangGraph** для более гибкого управления провайдерами.

## Технологии

- **Python 3.11+**
- **FastAPI** — веб-фреймворк
- **LangChain** — интеграция с LLM (Cerebras, OpenRouter)
- **Pydantic Settings** — управление конфигурацией
- **Uvicorn** — ASGI-сервер
- **Docker** — контейнеризация
- **aiofiles** — асинхронная работа с файлами
- **docx2txt** — извлечение текста из `.docx`

## API Эндпоинты

Все эндпоинты имеют префикс `/api/v1`.

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/` | Проверка работы сервиса (возвращает `"Hello from NeuroTest!"`) |
| `POST` | `/files` | Загрузка файла с тестом (`.docx`) |
| `POST` | `/files/json_text` | Создание JSON-файла с вопросами без ответов |
| `POST` | `/files/json_answer` | Создание JSON-файла с вопросами и правильными ответами |
| `GET` | `/files` | Получение списка файлов по типу (`docx`, `text`, `answer`) |

## Установка и запуск

### Локальный запуск

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/TigranAko/NeuroTest.git
   cd NeuroTest/backend
   ```

2. Создайте файл `.env` в корневой директории (рядом с `backend/`) со следующими переменными:
   ```env
   TAVILY_API_KEY=your_tavily_key
   CEREBRAS_API_KEY=your_cerebras_key
   OPENROUTER_API_KEY=your_openrouter_key
   ```

3. Установите зависимости (рекомендуется использовать `uv`):
   ```bash
   uv sync
   ```

4. Запустите сервер:
   ```bash
   python main.py
   ```
   Сервер будет доступен по адресу `http://localhost:8000`.

### Запуск в Docker

1. Соберите образ:
   ```bash
   docker build -t neurotest-backend ./backend
   ```

2. Запустите контейнер:
   ```bash
   docker run -p 8000:8000 --env-file .env neurotest-backend
   ```

## Переменные окружения

| Переменная | Описание |
|------------|----------|
| `TAVILY_API_KEY` | API-ключ для Tavily Search (поиск информации для проверки ответов) |
| `CEREBRAS_API_KEY` | API-ключ для Cerebras (генерация ответов) |
| `OPENROUTER_API_KEY` | API-ключ для OpenRouter (парсинг текста теста) |

## Особенности реализации

- **Парсинг тестов** (`text2json.py`): текст разбивается на чанки по 4000 символов, каждый чанк обрабатывается LLM через OpenRouter для выделения вопросов и вариантов ответов.
- **Генерация ответов** (`json2answer.py`): для каждого вопроса выполняется поиск через Tavily, результаты передаются в Cerebras для определения правильных вариантов.
- **Файловое хранилище**: все загруженные файлы и сгенерированные JSON сохраняются в директорию `backend/files/`.
